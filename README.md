## Branching Strategy
- `main`: Production-ready branch
- `dev`: Integration branch
- `feature/login`: Feature development branch

## Workflow
1. Feature created on `feature/*` branch.
2. Merged into `dev` via Pull Request.
3. After testing, merged into `main`.

- Branches
- Pull Requests
- Tags
- Commit history
- .gitignore
